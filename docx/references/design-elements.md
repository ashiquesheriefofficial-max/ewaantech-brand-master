# Design Elements — Proposal Template Shapes (Revised 2026)

The Ewaantech proposal template carries two recurring brand graphics. Each is generated as a transparent PNG and anchored to the page (not a paragraph) so it renders identically in Word, LibreOffice, and PDF.

> **Revision note.** The earlier design (V-bottom flag + flat strip + bottom-right corner triangle) has been superseded. The corner triangle is no longer used. The flag and strip have new geometry per the revised reference doc.

---

## 1. Top-left hexagonal flag

A vertical red shape bleeding from the page top-left. The **left edge** runs the full height of the shape; the **right edge** is shorter, with both right corners chamfered by diagonal cuts. The result looks like a flag/banner with its right edge tapered into a soft point.

| Spec | Cover version | Content-page version |
|---|---|---|
| Source PNG size | 60 × 303 px | 60 × 303 px (same source, scaled smaller) |
| Angle depth (chamfer reach) | 60 px | 60 px |
| Fill | `#CE202F` | `#CE202F` |
| Render size in docx | 42 × 212 px | 38 × 150 px |
| Anchor — horizontal | Page, offset 0 (flush left) | Page, offset 0 |
| Anchor — vertical | Page, offset 0 (flush top) | Page, offset 0 |
| Behind document text | true | true |

**Polygon vertices** (clockwise, in the source PNG coordinate space `60 × 303` with `angle_depth = 60`):

```
(0, 0)                          ← top-left point
(width, angle_depth)            ← end of top chamfer (= 60, 60)
(width, height - angle_depth)   ← start of bottom chamfer (= 60, 243)
(0, height)                     ← bottom-left point (= 0, 303)
```

Cover version is taller because it carries more visual weight; content-page version is shorter so the section title block clears it.

## 2. Bottom tapered strip

A thin horizontal red bar running just above the foot of every page. The **bottom edge** is flat across the full width; the **top edge** tapers diagonally to a single-pixel point at each end. Looks like an elongated banner laid flat.

| Spec | Value |
|---|---|
| Source PNG size | 1080 × 11 px |
| Taper length (each end) | 10 px |
| Fill | `#CE202F` |
| Render size in docx | 820 × 8 px (~8.5 × 0.087 in) |
| Anchor — horizontal | Page, offset 38100 EMU (~0.04 in inset from page-left, so the left taper is visible) |
| Anchor — vertical | Page, offset 10622280 EMU (~11.6 in from top — just above A4 page foot) |
| Behind document text | false |

**Polygon vertices** (clockwise, in the source PNG coordinate space `1080 × 11`):

```
(0, height-1)            ← bottom-left point
(taper, 0)               ← top-left corner of full-thickness section
(width - taper, 0)       ← top-right corner of full-thickness section
(width-1, height-1)      ← bottom-right point
```

This strip is also where the brand-red page number sits in the content-page footer, right-aligned just above the bar.

## 3. Section marker (small red square) — fallback only

A small red square that pairs with the section title block on layouts where the top-left flag is suppressed. Not used in the standard proposal-template layout (the flag handles this duty) but kept available as a fallback.

| Spec | Value |
|---|---|
| Width (px source) | 70 |
| Height (px source) | 200 |
| Fill | `#CE202F` |

## ~~4. Bottom-right corner triangle~~ — REMOVED

The earlier design carried a small red triangle in the bottom-right corner of the cover. The revised reference doc drops this element entirely. **Do not generate or place it.** The cover now closes visually with just the tapered bottom strip.

## Generation

Both shapes are generated by `scripts/make_graphics.py` using Pillow. Run it once in the working directory before any build:

```bash
cd /home/claude/work
python3 $SKILL_PATH/scripts/make_graphics.py
```

This produces:

```
top_left_flag.png
bottom_strip.png
section_marker.png
```

The script reads the brand red from a constant at the top — if the brand evolves to a different red, update the script (and `references/brand-system.md`).

## Anchoring in docx-js

Floating images go in a `Header` because that's the only place where docx-js will let a `floating` image anchor to the page rather than to a paragraph. The header itself can be empty otherwise (or carry the logo). Example:

```js
const contentHeader = new Header({
  children: [new Paragraph({ children: [contentTopFlag(), logoFloater()] })],
});
```

Where `contentTopFlag()` and `logoFloater()` return `ImageRun` instances with:

```js
floating: {
  horizontalPosition: { relative: HorizontalPositionRelativeFrom.PAGE, offset: ... },
  verticalPosition:   { relative: VerticalPositionRelativeFrom.PAGE, offset: ... },
  wrap: { type: TextWrappingType.NONE },
  behindDocument: true,   // true for the flag, false for the strip and logo
}
```

Offsets are in EMU (1 inch = 914400 EMU). The `lib.js` helpers encode the correct offsets for an A4 page — don't recompute them every build.
